## Быстрый старт (локально)

### 1. Зависимости

```bash
poetry install # бэк
npm install    # фронт
```

### 2. Переменные окружения

```bash
cp .env-example .env
# Заполните BOT_TOKEN
```

### 3. Redis

Для локального запуска подойдет [memurai](https://www.memurai.com/).
Просто установить куда угодно и запустить memurai.exe

Запуск через докер:

```bash
docker compose up redis -d
```

### 4. Запуск сервисов (3 терминала) 

```bash
# Терминал 1 — бот
poetry run python main.py -b

# Терминал 2 — ML worker
poetry run python main.py -w

# Терминал 3 — API
poetry run python main.py -a

# Терминал 4 — Фронтенд
cd frontend_charts
npm start
```

### 5. Работа моделей

Создание и использование моделей всецело опирается на конфиг файл `packages/shared/config.py` на объекты `LABELS` и `LABELS_RELATIONS`

Запуск обучения моделей
```bash
# -t --train
poetry run python main.py -t
```

По умолчанию для обучения используется файл `data/excel/data_ml.xlsx`, но можно задать иной путь
```bash
# -f --file
poetry run python main.py -t -f path/to/file.xlsx
```

Формат Excel файла:

| text | label1 | label1 | label1 | ... |
|:----:|:------:|:------:|:------:|:---:|
| сообщение | класс | класс | класс | ... |

Первый столбец всегда содержит сообщения и называется text, все последующие столбцы содержат разметку. Заголовки столбцов после первого должны совпадать с метками LABELS в конфиге

По завершении программы будут созданы модели по тем меткам, которые были перечислены в конфиге **И** присутствовали в табличке
```python
# Если нужной модели нет здесь или в табличке нет столбца с таким заголовком, модель не будет создана
# Если в таблице присутствует класс, не указанный здесь, обучение остановится, выбросив исключение
# Первый столбец таблицы всегда рассматривается как столбец с сообщениями.
# Порядок определения моделей и порядок столбцов не важны, важно точное совпадение названий между столбцами таблицы и названиями моделей

LABELS = {
    "useful": ["да", "нет"],
    "sentiment": ["positive", "negative", "neutral"],
    "category": ["окна", "парковка", "пожарная сигнализация", "лифт", "кондиционер", "ук", "улица", "соседи", "животные", "вода/отопление", "потеря", "мусор", "посторонние", "запах", "электричество", "водители", "охрана", "канализация"],
}
```

По умолчанию модель предсказывает абсолютно каждое приходящее сообщение. LABELS_RELATIONS позволяет формировать предсказания, основываясь на предсказаниях другой модели.

```python
# Зависимость классификатора "category" от классификатора "useful"
# "category" будет размечать только сообщения, помеченный классификатором "useful" как 0 (полезное)

LABELS_RELATIONS = {
    "category": {"based_on": "useful", "filtering_label": 0}
}
```

ВАЖНО — модель, базирующаяся на другой модели, должна обязательно идти после нее в конфиге, где именно - не важно, главное, чтобы не до.

В ином случае при переборе моделей и формировании предсказаний опирающаяся модель будет стараться получить предикты опираемой модели до того, как они были сформированы

```python
# category зависит от useful и определяется после нее

LABELS = {
    "useful": ["да", "нет"],
    "sentiment": ["positive", "negative", "neutral"],
    "category": ["окна", "парковка", "пожарная сигнализация", "лифт", "кондиционер", "ук", "улица", "соседи", "животные", "вода/отопление", "потеря", "мусор", "посторонние", "запах", "электричество", "водители", "охрана", "канализация"],
}
```


### 6. Аргументы
- `-t, --train` — запустить обучение моделей
- `-f, --file` — путь к файлу с данными (опционально, ничего не сделает без `-t`)
- `-w, --worker` — запуск сервиса классификации сообщений
- `-a, --api` — запуск сервиса API для фронтенда
- `-b, --bot` — запуск сервиса с Telegram ботов

Пожалуйста, не смешивайте аргументы (кроме `-t` и `-f`), технически оно должно работать, но получится амальгама из всех сервисов в одном терминале
